Liquibase MongoDB – Required Files & Libraries
1. Core Components

Regardless of OSS or Commercial, you need:

Liquibase CLI / Core Library

The main Liquibase executable (liquibase CLI or Java JAR).

Provides the command framework, changelog parsing, and change execution.

Liquibase MongoDB Extension

A database-specific extension that lets Liquibase talk to MongoDB.

Distributed as a JAR file.

2. OSS Setup (Open Source)
Required JARs / Files

liquibase-core (comes with CLI or Maven/Gradle dep)
→ Base Liquibase engine.

liquibase-mongodb extension (OSS)
→ Provides support for MongoDB changes (insertOne, insertMany, update, delete, runCommand).

MongoDB Java Driver
→ Required for connecting to MongoDB. Typically bundled with the extension.

How to Install

Option A: Standalone CLI
Download the extension JAR and put it in the liquibase/lib folder next to the Liquibase executable:
liquibase/
  ├─ liquibase (CLI binary or liquibase.jar)
  ├─ lib/
      ├─ liquibase-mongodb-<version>.jar
      ├─ mongodb-driver-sync-<version>.jar

Option B: Maven/Gradle Dependency (Java project)
<dependency>
  <groupId>org.liquibase.ext</groupId>
  <artifactId>liquibase-mongodb</artifactId>
  <version>4.30.0</version> <!-- match your Liquibase version -->
</dependency>

✅ OSS MongoDB extension allows only basic changesets:

insertOne

insertMany

update

delete

runCommand


3. Commercial Setup (Pro/Enterprise)
Additional JARs / Files

In addition to the OSS libraries, Commercial versions provide Pro extensions:

liquibase-commercial.jar
→ Core library enabling enterprise-only changes and advanced rollback.

liquibase-mongodb-pro.jar (sometimes bundled into the commercial package)
→ Unlocks advanced MongoDB features, e.g.:

mongoFile changesets (executing external .js or .json scripts).

Extended rollback support.

Schema diffing and audit/reporting for MongoDB.

License file (liquibase.lic)
→ Must be placed in $HOME/.liquibase/ or set via LIQUIBASE_LICENSE_KEY environment variable.

Folder Structure Example

liquibase/
  ├─ liquibase (CLI)
  ├─ lib/
      ├─ liquibase-mongodb-<version>.jar
      ├─ liquibase-commercial-<version>.jar
      ├─ liquibase-mongodb-pro-<version>.jar
      ├─ mongodb-driver-sync-<version>.jar
  ├─ liquibase.lic   <-- License file




Summary:

OSS = liquibase-core + liquibase-mongodb + MongoDB driver.

Pro = OSS libs + liquibase-commercial.jar + liquibase-mongodb-pro.jar + license file.

Key difference: Pro supports mongoFile changesets and advanced enterprise features, while OSS only supports basic MongoDB operations (insertOne, insertMany, update, etc.).
